## Three-Tier Pipeline

| Tier | What | Example |
|------|------|---------|
| **1 — Raw** | Original artifact, unmodified | PDF chapter, SDK header, source file |
| **2 — Searchable** | Markdown conversion, **faithful to original text** | `pdftotext` + cleanup, or model-assisted |
| **3 — Field Guide** | Condensed card: key tables, 5 registers, 3 failure modes | Hand-written after reading |

**Rule: Tier 2 from Tier 1, not from re-fetch. Tier 3 from Tier 2.**

### Tier 2 fidelity standard

Tier 2 is a **format conversion**, not a rewrite. The goal is the same content in
a searchable, renderable format.

- **Prose**: Preserve the original wording. Don't summarize paragraphs into
  bullet points — that loses detail and belongs in Tier 3.
- **Tables**: Reconstruct as markdown tables. These may require model-assisted
  reconstruction since `pdftotext` loses column alignment.
- **Headings**: Map to markdown `##`/`###` hierarchy matching the original
  numbering (1.1 → `## 1.1.`, 1.2.3 → `### 1.2.3.`).
- **Figures/diagrams**: Note their existence and what they depict, but don't
  attempt to reproduce them. Use a blockquote: `> **Figure N** — description. See source PDF page X.`
- **Formatting improvements** are fine: fixing line breaks that `pdftotext`
  introduced mid-sentence, removing page headers/footers, cleaning up
  whitespace. The text itself stays the same.
- **Metadata header**: Each Tier 2 file should start with source path,
  page spans (both printed and physical), conversion method, and loss notes.

## Conversion Recipes

### Datasheet PDF → Tier 2 Markdown

This datasheet was generated by **Asciidoctor PDF** (not scanned), so text
extraction is clean. Tables may lose formatting but prose and headings survive.

Splitter utility (preferred):
```bash
./split-datasheet.py [source.pdf] [output_dir]
```

Fallback/reference splitter:
```bash
./split-datasheet.sh [source.pdf] [output_dir]
```

Equivalent rewrites (same logic, different language ergonomics):
```bash
./run-split-go.sh [source.pdf] [output_dir]
./run-split-rs.sh [source.pdf] [output_dir]
./run-split-clj.sh [source.pdf] [output_dir]
```

**Recipe A — fast pass (good enough for most chapters):**
```bash
pdftotext -layout chapter.pdf chapter-raw.txt
# Then clean up: strip headers/footers, fix section headings, add markdown ##
```

**Recipe B — model-assisted (for chapters with important tables):**
1. Extract text with `pdftotext -layout`
2. Read the PDF pages directly (Claude can read PDFs)
3. Reconstruct tables as markdown tables
4. Flag any diagrams that are lost (note what's on which page)

**Recipe C — targeted register extraction (for 09b etc.):**
Don't convert the whole file. Extract just the registers you need:
```bash
pdftotext -f <start> -l <end> -layout chapter.pdf /dev/stdout | grep -A 20 "GPIOx_CTRL"
```

### Source Code → Tier 2

Source files are already Tier 2 (they're text). Tier 3 for source is an
**annotated reading guide**: key functions, what they do, which registers
they touch, and how they map to the datasheet.

## Cost Lessons (from DiT project)

- **Opus reading PDF pages is expensive.** 3 papers (45 pages total) burned
  45% of a 5-hour Claude budget. For bulk Tier 2 conversion, use `pdftotext`
  first and only bring in a model for tables/equations that need reconstruction.
- **Don't collapse Tier 1 and Tier 2.** Keep the raw artifact. You'll re-extract
  differently later as your questions change.
- **Tier 3 is where the learning happens.** Tier 2 is just making things searchable.
  The real value is writing the condensed field guide yourself — that's where
  you internalize the material.

## Tier 3 Field Guide Template

When you write a field guide for a chapter, use this shape:

```markdown
# [Chapter Name] — Field Guide

## In One Sentence
What this peripheral/subsystem does and why you care.

## Key Concepts
3-5 bullet points. The things you need to hold in your head.

## The Registers That Matter
| Register | Offset | What it controls |
|----------|--------|-----------------|
| ... | ... | ... |

## Common Failure Modes
1. [Symptom] → [Cause] → [Fix]
2. ...
3. ...

## Gotchas
Things that surprised you or are easy to get wrong.

## Cross-References
- Datasheet: page X
- C SDK: `hardware_foo/foo.c` line Y
- Embassy: `foo.rs` line Z
```
